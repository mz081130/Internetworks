HTTP & HTTPS- What is the Internet

HTTP

Hyper Text Transfer Protocol

It’s a protocol to transfer resources to and from the server, such as HTML document. When we open a webpage bunch of “get” requests are send by the web browser to the server via the internet. A single web page have multiple get requests, for example on how the web page looks, get request for images and videos. A client can also send data to the server, for example data generated when a user sign-up for an account, these request are called post request.

HTTP uses TCP (Transmission Control Protocol), generally over port 80, to send and receive data packets over the web. To put it simply it is a protocol that's used by a client and server which allows you to communicate with other websites. The client sends a request message to an HTTP server (after the TCP handshake) which hosts a website, the server then replies with the response message. The response message contains completion status information, such as HTTP/1.1 200 OK

HTTPS

Hyper Text Transfer Protocol Secure

It’s a secure way of HTTP communication between a client and a server. In HTTP, communication can be compromised, therefore an additional layer is added via SSL (Secure Socket layer) or TLS (Transport layer security).

Generally sites running over HTTPS will have a redirect in place so even if you type in http:// it will redirect to deliver over a secured connection. HTTPS also uses TCP (Transmission Control Protocol) to send and receive data packets, but it does so over port 443, within a connection encrypted by Transport Layer Security (TLS)

HTTPS digital certificates are issued by entrusted entities that verify the identity of website and are presented to the user in the form of a lock next to the HTTPS://.
